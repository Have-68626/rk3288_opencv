# 相机调用全链路审查与优化 Spec

## Why
当前项目相机调用链路涉及权限、初始化、预览帧回调、拍照与录像、多线程与资源释放等复杂环节，容易出现合规缺失、内存/句柄泄漏、预览阻塞、异常无法恢复与兼容性问题。本变更以“可复现的审查报告 + 可验证的优化落地”为目标，对相机全链路进行全面审查与优化。

## What Changes
- 审查并完善相机权限申请与合规流程（首次申请、拒绝后引导、仅在需要时申请、前后台限制）。
- 审查并修复相机初始化/释放逻辑，确保无泄漏、无重复打开、生命周期切换安全。
- 审查并优化预览帧回调处理，避免主线程阻塞，确保背压策略与丢帧策略明确且可控。
- 审查拍照与录像功能稳定性：状态机完整、资源占用可控、异常可恢复、重复操作健壮。
- 审查相机参数配置合理性：分辨率/帧率/对焦/曝光/闪光/方向/多摄选择，按能力查询适配。
- 补齐错误处理与异常恢复机制：可观测（日志/指标）、可重试（有限次数/退避）、可降级（功能降级/重启会话）。
- 审查多线程并发访问安全：串行化相机控制面、隔离耗时任务、避免竞态与死锁。
- 审查内存占用：预览缓冲复用、图片/视频缓存限制、避免频繁分配与大对象泄漏。
- 输出详细检查报告与改进方案（问题清单、风险评级、复现步骤、修复建议、验收方法）。

## Impact
- Affected specs: 相机权限与合规、相机生命周期管理、预览帧处理性能、拍照/录像稳定性、异常恢复、多线程与内存优化、兼容性测试
- Affected code（预期范围，实际以审查结果为准）:
  - `app/src/main/java/**`（Android UI 与 Camera2/CameraX 调用）
  - `src/java/**`（若仍存在并参与构建的 Java 逻辑）
  - `app/src/main/AndroidManifest.xml`
  - `app/src/main/res/**`（权限提示/设置引导 UI 资源）
  - `app/src/main/cpp/**`（JNI/Native 处理链路，如参与预览帧处理）
  - `docs/**` 与 `DEVELOP.md`（流程与兼容性记录）

## ADDED Requirements
### Requirement: 权限申请与合规
系统 SHALL 在相机功能启动前完成必要权限检查与申请，并对拒绝/永久拒绝提供明确的用户引导。

#### Scenario: 首次进入相机功能
- **WHEN** 用户首次进入相机页面/触发相机启动
- **THEN** 系统仅申请必要权限（相机/音频/存储按需）
- **AND** 权限未授予时不启动相机硬件访问

#### Scenario: 权限被拒绝或永久拒绝
- **WHEN** 用户拒绝权限或勾选“不再询问”
- **THEN** 系统展示可理解的原因说明与设置跳转入口
- **AND** 不进入异常循环请求

### Requirement: 生命周期与资源释放
系统 SHALL 在页面切换、后台/前台、旋转、进程重建等情况下，保证相机资源正确释放与可重新初始化。

#### Scenario: 前后台切换
- **WHEN** App 进入后台
- **THEN** 相机会话与相关资源被关闭并释放
- **WHEN** App 回到前台且权限满足
- **THEN** 相机可恢复到可用状态且不崩溃

### Requirement: 预览帧回调无阻塞
系统 SHALL 保证预览帧回调不会阻塞主线程，并提供明确的背压策略（丢帧/队列上限/单帧最新）。

#### Scenario: 高帧率预览
- **WHEN** 预览帧持续到达（例如 30fps）
- **THEN** 回调处理不在主线程执行重计算
- **AND** 队列长度受限且不会无限增长

### Requirement: 拍照与录像稳定性
系统 SHALL 具备明确的状态机与错误恢复能力，避免重复点击导致崩溃或资源泄漏。

#### Scenario: 连续拍照/开始停止录像
- **WHEN** 用户快速连续触发拍照或开始/停止录像
- **THEN** 系统状态机保持一致，不出现死锁/崩溃
- **AND** 错误发生时给出可恢复路径（重试/重建会话/提示用户）

### Requirement: 参数配置按能力适配
系统 SHALL 基于设备能力查询选择合理参数（分辨率/帧率/对焦/曝光/闪光/方向/多摄选择），避免硬编码导致兼容性问题。

#### Scenario: 多设备兼容
- **WHEN** 设备不支持指定分辨率/帧率/对焦模式
- **THEN** 系统自动降级到可用参数并记录可诊断日志

## MODIFIED Requirements
### Requirement: 日志与可观测性
相机链路关键状态变化与错误 SHALL 被记录（不包含敏感数据），并可关联到一次会话（session）用于追踪问题。

## REMOVED Requirements
### Requirement: 无
**Reason**: 以审查与优化为主，不移除既有业务能力。
**Migration**: 保持对外行为兼容，优化为内部实现与健壮性增强。
