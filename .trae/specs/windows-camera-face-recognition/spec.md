# Windows 摄像头人脸识别测试系统 Spec

## Why
项目需要在 Windows 10/11（64 位）上提供可复现、可量化的人脸识别测试系统，用于验证摄像头接入、实时预览、多脸识别、日志与性能指标，并形成稳定的交付与回归基线。

## What Changes
- 新增 Windows 摄像头接入模块：基于 Media Foundation（优先）或 DirectShow 进行设备枚举、连接与分辨率配置。
- 新增 Windows 原生窗口预览应用：基于 Win32 API（单语言 C++）实现实时预览、相机切换、分辨率调整、画面翻转等控制。
- 新增人脸识别核心模块：基于 OpenCV（优先）实现检测、对齐/预处理、特征提取、相似度比对、多人脸管理与阈值策略。
- 新增结果展示与日志：在预览画面实时叠加框/姓名/置信度，落盘结构化识别日志（CSV/JSON）。
- 新增测试与评估：单元测试覆盖核心算法与阈值逻辑；提供可复现的性能与光照条件评估报告模板与生成工具。
- 新增构建与使用文档：补齐 Windows 编译文档、依赖清单与用户操作手册（ZH_CN）。

## Impact
- Affected specs: Windows 摄像头采集、实时预览 UI、人脸识别评估、性能与兼容性验证、文档与合规
- Affected code（预期新增为主）:
  - `src/win/**`（Windows 专用 C++ 代码：采集/渲染/UI/识别）
  - `tests/win/**`（单元测试与性能评估）
  - `CMakeLists.txt`（增加 Windows 目标与选项）
  - `DEVELOP.md`、`README.md`、`CREDITS.md`（构建与依赖说明）

## ADDED Requirements
### Requirement: Windows 摄像头接入模块
系统 SHALL 提供摄像头设备枚举与连接能力，并支持 USB 摄像头与内置摄像头的自动识别与分辨率配置。

#### Scenario: 枚举与连接
- **WHEN** 用户启动 Windows 测试程序
- **THEN** 程序展示可用摄像头列表（名称、设备 ID、支持分辨率/帧率集合）
- **AND** 用户选择设备后可成功打开并开始采集

#### Scenario: 权限与隐私设置
- **WHEN** 系统层面禁用摄像头权限或设备被占用
- **THEN** 程序给出明确错误分类与恢复建议（Windows 隐私设置入口提示/释放占用提示）
- **AND** 不发生崩溃与资源泄漏

### Requirement: 实时视频流预览
系统 SHALL 提供 Win32 原生窗口预览，满足 30FPS 以上流畅预览并支持 640x480 至 1920x1080 分辨率。

#### Scenario: 预览与控制
- **WHEN** 摄像头开始采集
- **THEN** 窗口持续渲染预览画面，帧率统计可见
- **AND** 支持在运行时切换摄像头、切换分辨率、水平/垂直翻转

### Requirement: 人脸识别核心流程
系统 SHALL 实现完整的人脸识别流程：检测 -> 特征提取 -> 比对 -> 输出身份信息，支持单人/多人脸同时识别。

#### Scenario: 多人脸识别
- **WHEN** 画面出现多张人脸
- **THEN** 对每张人脸输出独立的框、ID/姓名、置信度
- **AND** 多人脸之间不会相互覆盖导致结果错配

### Requirement: 识别结果展示与日志
系统 SHALL 在 UI 叠加识别结果，并将识别事件落盘为结构化日志，字段至少包括：
- 时间戳
- 摄像头 ID/名称
- 人脸数量
- 每张人脸的框坐标、识别结果（ID/姓名）、置信度/距离
- 错误码与错误原因（如有）

#### Scenario: 识别日志可审计
- **WHEN** 程序运行并发生识别事件
- **THEN** 日志文件持续追加记录且可被离线解析

### Requirement: 性能与资源约束
系统 SHALL 在 Windows 10/11 64 位下满足：
- 内存占用不超过 500MB
- CPU 使用率控制在 30% 以内（以指定测试脚本与条件为准）
- 具备完整异常处理机制，长时间运行稳定

#### Scenario: 长时间稳定性
- **WHEN** 程序连续运行 ≥ 30 分钟（预览+识别开启）
- **THEN** 无崩溃、无明显内存泄漏、帧率波动可接受并可在报告中量化

### Requirement: 准确率指标（≥95%）
系统 SHALL 提供“可复现的评估数据集与评估脚本”，并在定义的评估口径下达到识别准确率 ≥ 95%。

#### Scenario: 离线评估
- **WHEN** 使用评估工具在指定数据集上运行
- **THEN** 输出总体准确率、误识率、拒识率、混淆矩阵（如适用）
- **AND** 若 <95%，输出可执行的调优方案（阈值/预处理/模型/采集策略）

### Requirement: 测试与交付
系统 SHALL 交付：
- 完整源代码
- 编译文档（Windows 10/11 x64）
- 依赖库列表与许可证声明（更新 `CREDITS.md`）
- 用户操作手册（ZH_CN）
- 单元测试与性能测试报告（含不同光照条件）

## MODIFIED Requirements
### Requirement: 便携性与配置管理
系统 SHALL 禁止硬编码依赖路径，改用环境变量/配置文件（例如 `.env` 或 `.ini`），并在文档中提供一键复现指南。
