# Android 摄像头与人脸识别调研文档 Spec

## Why
当前项目需要在 RK3288 及 Android 设备上稳定、低延迟地获取摄像头数据并进行端侧生物识别。为降低集成风险与后期返工，需要一份覆盖 API 选型、生命周期、权限、性能基准、测试与合规的系统性调研文档，沉淀为可复用的工程规范。

## What Changes
- 在 DEVELOP.md 中新增两大研究章节：
  - “第一部分：Android APP 摄像头调用机制研究”
  - “第二部分：人脸识别技术实现方案研究”
- 为每个章节补充：对比表、时序图/架构图、关键代码片段、异常处理策略、性能基准与可复现测试方案。
- 在 DEVELOP.md 增加“图表索引”“参考文献”“风险清单”“修订记录”，并更新版本号（遵循既有版本字段格式）。
- **BREAKING**：若现有 DEVELOP.md 章节编号需要重排以保证连续性，将同步调整章节号与目录索引（内容语义不变）。

## Impact
- Affected specs: 文档规范、Android Camera 调用规范、端侧人脸识别选型与合规、测试基准与验收口径
- Affected code: 仅文档与脚本模板（不直接改动业务代码）；可能新增 `scripts/` 下的示例脚本片段（若采用“可复现脚本”交付）
- Affected files:
  - DEVELOP.md（主交付）
  - （可选）scripts/ 下用于基准测试与自动化验证的脚本模板引用

## ADDED Requirements

### Requirement: 摄像头调用机制研究章节
系统 SHALL 在 DEVELOP.md 中提供完整且可执行的“Android APP 摄像头调用机制研究”，至少包含下列内容：
- Camera1/Camera2/CameraX 与典型第三方封装库的差异、适用场景、版本兼容性与维护状态评估。
- “主动调用”与“系统回调”两种模式的生命周期对比、权限模型与性能开销（含线程/回调队列说明）。
- 摄像头枚举与能力查询：前置/后置/广角/TOF/红外等类型的识别方法；闪光灯、对焦、曝光、FPS、输出格式、硬件级别（LEGACY/FULL/LEVEL_3）判定流程。
- 运行时权限：Manifest 声明 + 动态申请 + Android 13+ 细粒度权限 + 后台摄像头限制；必须提供“拒绝/永久拒绝/跳转设置页返回”的完整处理模板。
- 完整时序图：打开 → 预览配置（Surface/SurfaceTexture/PreviewView）→ 拍照/录像 → 回调数据（YUV_420_888/JPEG/MP4）→ 释放资源；并明确标注常见泄漏点与 CameraService 重启风险及检测手段。
- 性能基准与脚本：冷启动首帧 < 600ms、30 次连续拍照 OOM=0、后台切换 50 次 CameraService 重启=0；提供通过标准与采集指标方式。

#### Scenario: 成功
- **WHEN** 开发者按文档集成 CameraX（或 Camera2）并运行基准脚本
- **THEN** 可在指定设备上复现首帧与稳定性指标，并能从日志/报告定位异常

#### Scenario: 权限异常
- **WHEN** 用户拒绝权限或永久拒绝并从设置页返回
- **THEN** 应用能给出明确 UI 引导与重试路径，且不会触发相机资源泄漏

### Requirement: 人脸识别方案研究章节
系统 SHALL 在 DEVELOP.md 中提供完整且可执行的“人脸识别技术实现方案研究”，至少包含下列内容：
- 离线端侧方案与云端方案汇总：识别原理、特征维度/模板大小、阈值/相似度口径、延迟指标与适用场景（含隐私与网络依赖差异）。
- Android 集成步骤：Gradle 依赖、SO/模型文件、授权激活、最小权限、混淆配置、ABI 过滤规则（按方案分别给出最小模板）。
- 硬件要求与基准：CPU 架构、NPU/GPU 加速、摄像头分辨率、RAM/存储；提供 1:N≤10000 的单帧比对 <150ms、识别率≥97% 的测量口径与数据记录格式。
- 活体检测对比：RGB 静默/红外/3D 结构光/动作指令；结合 ISO 30107-3 安全等级与防攻击用例（照片/视频/面具/翻拍）。
- 合规与优化：特征加密存储（AES-256-GCM）、模型热更新、多线程推理、CameraX Preview+ImageAnalysis 零拷贝、动态降级策略。
- 可复现 Demo 交付：Demo APK 构建脚本（Gradle 7.5 + NDK 25）、单测（JUnit+Robolectric）、集测（Espresso+MockCamera）、CI（GitHub Actions）自动验证识别率与内存泄漏指标的方案与最小配置模板。

#### Scenario: 端侧识别成功
- **WHEN** 设备满足最低硬件要求并按文档集成端侧方案
- **THEN** 可在本地离线完成检测/建库/1:N 比对并输出达标报告

#### Scenario: 云端识别降级
- **WHEN** 网络不可用或云端 API 失败
- **THEN** 应用能回退到本地检测或给出明确失败策略（不泄露生物特征数据）

### Requirement: 文档结构与可维护性
系统 SHALL 确保 DEVELOP.md：
- 章节编号连续；图表有编号与索引；表格字段定义一致。
- 所有代码片段具备可复制性，包含必要 import/依赖声明与异常处理分支。
- 引用与来源可追溯；涉及第三方 SDK/模型的许可证与合规注意事项有明确提示。
- 不硬编码本机路径；以环境变量/占位符方式表达路径与密钥配置。
