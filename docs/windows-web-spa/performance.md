# 性能验收口径与测量方法

## 指标
- 首屏加载 < 800ms：从浏览器导航开始到“首屏可交互的关键 UI 出现”
- 交互响应 < 100ms：常见操作（切换页面/打开设置/保存设置后 UI 更新）
- 内存占用 < 150MB：本地服务进程 + 浏览器页签合计（以本文口径为准）

## 测量前置
1) 使用 Release 构建运行本地服务：`win_local_service.exe`
2) 使用 Chrome/Edge 最新稳定版，关闭多余扩展
3) 清空缓存并强制刷新（Ctrl+Shift+R）

## 首屏加载（建议口径）
1) 打开 DevTools → Performance
2) 勾选 “Screenshots”
3) 点击 Record 后在地址栏打开：`http://127.0.0.1:<port>/`
4) 停止录制，读取：
   - 首次内容渲染（FCP）
   - 最大内容绘制（LCP）
   - 可交互点（以 UI 可点击并完成一次路由跳转为准）

## 交互响应
1) 在页面内执行：
   - 点击左侧“概览/预览/设置”切换路由
   - 在设置页点击“保存本地偏好”
2) DevTools → Performance 中观察事件处理耗时（main thread 任务）

## 内存占用
1) Windows 任务管理器：
   - 记录 `win_local_service.exe` 的内存
   - 记录浏览器进程中对应页签的内存（可用 Edge “浏览器任务管理器”查看更精确）
2) 两者相加作为验收口径
