## 3. 测试方法
### 3.1 性能指标口径
- FPS：窗口显示值（1 秒滑动统计）
- 帧时间：从 `render_metrics.jsonl` 读取（P50/P95/P99）
- CPU：使用任务管理器（按进程）或 PerfMon 记录
- 内存/句柄：从 `render_metrics.jsonl` 读取（rss_bytes/handle_count）或用任务管理器核对
- GPU 占用率：可采则采（本项目默认记录“显存占用 best-effort”，见 `render_metrics.jsonl` 的 gpu_*_bytes）
- 时长：
  - 基准：连续运行 ≥ 30 分钟，预览 + 识别开启
  - 长稳：72h 压测模式运行并落盘完整日志

### 3.2 光照条件
至少覆盖 3 组条件，并记录环境描述：
- 强光（靠窗/室外）
- 普通室内灯光
- 低照度（夜晚/弱光）

## 5. 结论与改进建议
- 是否满足：FPS≥30 / CPU≤30% / 内存≤500MB / 30分钟稳定运行
- 若不满足，建议改进方向：
  - 降低检测频率（例如每 N 帧检测一次，其他帧追踪）
  - 降低分辨率或提升 `min_face_size_px` 降低候选数量
  - 优化绘制与拷贝路径（减少 clone 与转换）
  - 引入更强的特征模型（DNN embedding）并使用更严格阈值
