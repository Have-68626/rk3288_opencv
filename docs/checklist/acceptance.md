# 项目验收清单 (Project Acceptance Checklist)

本清单用于 RK3288 OpenCV AI Engine 项目的开发、测试与上线阶段验收。所有检查项必须逐一确认并由相关责任人签字。

## 1. 开发阶段 (Development Phase)

### 1.1 代码规范与质量
- [ ] 所有 C++ 代码符合 C++17 标准，无 `-Wall` 编译警告。
- [ ] 核心类 (VideoManager, Engine, BioAuth) 包含完整的 Doxygen 风格注释。
- [ ] 无硬编码路径或魔术数字 (Magic Numbers)，所有配置项提取至 `Config.h` 或外部配置文件。
- [ ] 内存管理检查：无裸指针 `new`/`delete`，全面使用智能指针 (`std::shared_ptr`, `std::unique_ptr`)。
- [ ] 单元测试覆盖率达到 80% 以上 (使用 GTest)。

### 1.2 编译与构建
- [ ] CMake 构建系统在 Linux (Ubuntu 20.04+) 环境下一次性编译通过。
- [ ] Android NDK 交叉编译无报错，生成 `armeabi-v7a` 架构的 `.so` 库。
- [ ] 依赖库 (OpenCV, RKNN API, MPP) 版本正确且路径配置灵活。
- [ ] 提供一键编译脚本 (`build.sh`) 并验证通过。

### 1.3 功能实现
- [ ] **视频采集**: V4L2 能够稳定采集 720p@30fps 图像，无丢帧。
- [ ] **硬件解码**: RK MPP 能够正确解码 H.264 视频流，CPU 占用率 < 20%。
- [ ] **模型推理**: RKNN 加载模型成功，单帧推理耗时 < 50ms (MobileNet)。
- [ ] **显示输出**: DRM/KMS 双缓冲显示正常，无画面撕裂。

---

## 2. 测试阶段 (Testing Phase)

### 2.1 稳定性测试
- [ ] **长时间运行**: 连续运行 24 小时无崩溃，内存泄漏 < 1MB/小时。
- [ ] **开关机压力**: 连续重启设备 100 次，应用自动启动且功能正常。
- [ ] **热插拔测试**: 运行时插拔 USB 摄像头 50 次，程序能自动恢复或报错但不崩溃。
- [ ] **网络异常**: 断网/弱网环境下，本地功能不受影响，日志缓存正常。

### 2.2 性能测试
- [ ] **CPU 占用**: 满载运行 (采集+检测+显示) 时，CPU 总占用率 < 60% (4核 A17)。
- [ ] **内存占用**: 峰值内存占用 < 512MB。
- [ ] **启动速度**: 冷启动至首帧画面显示耗时 < 3秒。
- [ ] **端到端延迟**: 摄像头采集到 HDMI 显示延迟 < 100ms。

### 2.3 兼容性测试
- [ ] **RK3288 平台**: 在 Tinker Board / Firefly-RK3288 上验证通过。
- [ ] **Android 版本**: 兼容 Android 5.1, 7.1, 10.0。
- [ ] **摄像头兼容**: 支持 Logitech C270, 海康威视 USB 相机, MIPI CSI 相机。

---

## 3. 上线阶段 (Release Phase)

### 3.1 部署准备
- [ ] **发布包**: 生成包含 `app-release.apk`, `native-lib.so`, 模型文件,配置文件的发布包。
- [ ] **版本号**: `DEVELOP.md` 和 `build.gradle` 中的版本号已更新 (e.g., v1.3.0)。
- [ ] **Changelog**: `CHANGELOG.md` 已更新本次发布的内容。
- [ ] **签名**: APK 已使用正式密钥签名并对齐 (zipalign)。

### 3.2 文档与交付
- [ ] **用户手册**: 提供简明的用户操作指南 (PDF/Markdown)。
- [ ] **部署文档**: 提供详细的设备刷机、安装与配置步骤。
- [ ] **源码归档**: Git Tag 已打好，源码已推送到远程仓库。
- [ ] **已知问题**: 记录当前版本遗留的已知问题 (Known Issues) 及规避方案。
