## 4. 评分汇总

| 维度 | 均分 | 评语 |
|:----|:----:|:------|
| 代码可读性 | **2.8** | 新旧混合，新代码好但大文件拖低 |
| 模块化/耦合度 | **2.9** | 抽象接口好，Engine紧耦合+全局状态多 |
| 错误处理 | **2.9** | 错误字符串好但部分返回布尔值漠视故障 |
| 性能 | **3.2** | NEON向量化+reserve()+节流设计正确 |
| 可测试性 | **1.9** | **最大弱点** — 全局状态+无DI+静态缓存 |
| C++ 现代性 | **2.9** | unique_ptr/原子变量好，部分原始指针残留 |

### 总体评分：**中（2.8/5）**

**设计亮点**: FaceDetector/Embedder 抽象接口、ModelRegistry工厂、适配器模式、NEON向量化
**核心短板**: 可测试性极低（1.9/5）、Engine紧耦合、全局静态状态泛滥

## 5. 推荐重构

**P1**: 解耦 FaceInferStages 静态缓存 → ModelRegistry 托管实例
**P1**: Engine 依赖注入 → 构造函数接受 `unique_ptr<VideoManager>` 等
**P2**: 拆分 processFrame() → preprocessFrame/trackFaces/evaluateThrottle/renderResults/collectStats
**P2**: native-lib.cpp 全局状态 → JNI绑定类封装
**P3**: 消除 jsonEscape/nowEpochMillis 代码重复
