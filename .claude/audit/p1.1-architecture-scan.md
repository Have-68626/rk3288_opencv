**日期**: 2026-07-04

## 1. 项目架构总览

### 1.1 顶层目录

| 目录/文件 | 用途 |
|---|---|
| `app/` | Android 应用程序（Gradle 构建，含 Activity、布局、资源） |
| `src/cpp/` | 跨平台 C++ 核心引擎（视觉算法、推理管线、工具） |
| `src/win/` | Windows 平台本地服务（摄像头、HTTP 服务器、D3D 渲染、Win32 组件） |
| `src/java/` | Android Java 层（JNI 桥接、摄像头控制、UI Activity、日志） |
| `web/` | Web SPA 前端（React 18 + TypeScript + Vite + Ant Design 5） |
| `tests/` | 单元测试（cpp/、win/、fixtures/、unit/） |
| `docs/` | 设计文档、架构迁移、BSP 约束 |
| `config/` | Windows 配置 INI 文件 |
| `models/` | DNN 模型文件（仅 .gitkeep，模型需单独下载） |
| `deps/` | 第三方依赖（opencv 子构建、insightface、qualcomm_snpe、rk_mpp） |
| `scripts/` | 构建/CI 脚本 |
| `.github/workflows/` | CI 配置 |

### 1.2 总体架构风格

**单仓库、多平台、分层单体** 架构。核心 C++ 代码以不同配置链接到三个独立入口：

1. **Android JNI 库** (`native-lib`) — 加载到 Android Java 层
2. **Windows 本地服务** (`win_local_service`) — REST API + Web SPA 后端
3. **CLI 工具** (`rk3288_cli`, `inference_bench_cli`, `opencv_verify_cli`)

Web 前端是独立 React 应用，编译为静态文件，由 Windows 本地服务通过 civetweb 提供服务。

## 2. CMake 依赖分析

### 2.1 关键观察

**无 CMake 子目录**：所有目标在单个根 CMakeLists.txt（~1000 行）中定义，无子目录 CMakeLists.txt。唯一 `add_subdirectory` 用于构建 OpenCV 本身。

### 2.2 内部模块依赖图

```
RK3288_AI_Engine (project)
│
├── OpenCV (add_subdirectory)
│   ├── opencv_core, imgproc, video, objdetect, face, dnn, ml, photo, videoio, highgui, ...
│
├── ncnn (find_package / FetchContent, optional) ── ncnn::ncnn
├── rk_libyuv (FetchContent, optional, Android only)
│
├── native-lib (Android JNI)
│   ├── CORE_SOURCES（所有 src/cpp/src/*.cpp）
│   ├── src/cpp/native-lib.cpp
│   └── 3 个跨平台 win/ 文件
│
├── rk3288_cli (Host / Android CLI)
│   ├── CORE_SOURCES
│   ├── src/cpp/main.cpp
│   └── 3 个跨平台 win/ 文件
│
├── win_local_service (Windows HTTP 服务)
│   ├── src/win/src/*.cpp（26 个文件）
│   └── core: ArcFaceEmbedder.cpp, FileHash.cpp
│
├── win_camera_face_recognition (旧 Win32 UI)
├── inference_bench_cli
├── face_infer_unit_tests, core_unit_tests, win_unit_tests
├── ncnn_precision_test, win_face_eval_cli, win_face_bench_cli, win_face_database_perf
│
└── 所有目标应用 RK_COMMON_INCLUDES（src/cpp/include, adapters, src/win/include）
```

### 2.3 CMake 问题

1. **无库封装**：CORE_SOURCES 在 6+ 个目标中逐字复制，而非构建为 rk_core 库。导致重复编译、维护负担、无 ABI 边界
2. **跨平台文件放错位置**：src/win/src/FaceDetector.cpp、LbphEmbedder.cpp、DnnSsdFaceDetector.cpp 被 Android 目标包含但位于 win/ 下
3. **RK_COMMON_INCLUDES 事后应用**：包含路径为所有目标统一设置，不允许每个目标细粒度覆盖

## 5. 问题清单

### P1 — 架构问题

1. **单体 CMake：无库封装** — CORE_SOURCES 在 6+ 目标中逐字复制
2. **误放跨平台文件** — FaceDetector.cpp/LbphEmbedder.cpp/DnnSsdFaceDetector.cpp 在 src/win/ 下但被 Android 使用
3. **缺乏顶层架构图** — 55+ 头文件、32 Java 文件、多个 React 页面，无高层架构图

### P2 — 设计问题

4. **Engine.h 职责过重** — 采集+检测+认证+事件+节流+渲染，三层重构正确识别但未实施
5. **BioAuth 条件编译风险** — `__has_include(<opencv2/face.hpp>)` 导致编译时不报错、运行时静默失败
6. **VideoManager.h 包含 opencv2/opencv.hpp** — 引入数百个头文件，减慢构建

### P3 — 技术债务

7. **设计规范超前于实现** — 三层重构和人员注册系统设计已批准但未实现
8. **Android 未在 CMake 中管理** — Java/Gradle 在 CMake 视角不可见
9. **Config 命名空间冲突风险** — 全局 `Config` 命名空间
10. **命名空间不一致** — 全局/rk_win/rk_wcfr 混用
