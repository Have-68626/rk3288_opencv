## 2. HTTP API 契约一致性

### 关键发现：`kRoutes[]` 数组为死代码

HttpFacesServer.cpp 定义 `kRoutes[16]`（line 274-291），但 `handleApi()` 完全不用，直接委托 `registry_->dispatch()`（仅 8 端点）。7 个内联处理函数 + `kRoutes` 数组为死代码。

### 端点可用性矩阵

| 端点 | 预期 | 实际 | 状态 | 影响 |
|:-----|:----:|:----:|:----:|:-----|
| `/api/v1/actions/crypto/rotate` | 200 | **404** | 🔴 严重 | SettingsPage.tsx:544 调用，功能损坏 |
| `/api/v1/actions/privacy/open` | 200 | **404** | 🔴 严重 | PreviewPage.tsx:409 调用，功能损坏 |
| `/api/v1/preview.jpg` | 200 | **404** | 🟡 高 | 已文档化可能被 SPA 使用 |
| `/api/v1/acceleration` | 200 | **404** | 🟡 中 | 已实现但无法访问 |
| `/api/v1/openapi` | 200 | **404** | 🟡 中 | openapi.yaml 存在但端点 404 |
| `/api/v1/settings/validate` | — | **404** | 🟡 中 | paths.ts 定义但服务端未实现 |

### 配置字段类型差异

| 字段 | C++ 输出 | TypeScript 类型 | 状态 |
|:-----|:--------:|:--------------:|:----:|
| `recognition.arcFaceModelPath` | 必填输出 | **缺失** | 🔴 编译错误 |
| `model.int8Enabled` | 输出 | **缺失** | 🔴 PUT 回传可能重置 |
| `configuredPath`/`resolvedPath` | **已移除（安全原因）** | 非可选 string | 🔴 UI 显示空白 |
