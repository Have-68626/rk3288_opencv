## 3. Top 5 重点关注

**P1**: civetweb SSL 证书验证跳过（H1）— `SSL_VERIFY_NONE` 允许 MITM
**P2**: FFmpeg RTMP 命令注入（H2）— 输入未经白名单校验
**P3**: civetweb 不安全字符串函数（M2）— 200+ 处 sprintf/strcpy/strcat
**P4**: DEVELOP.md 凭据模板（M1）— 存在真实凭据被提交风险
**P5**: civetweb MD5 认证（M5）— HTTP Digest 使用已被认为不安全的 MD5

---

## 4. 总体评估

项目自研 C++ 代码质量良好：无原始指针数组问题，使用现代 C++ 模式（vector/string/智能指针/RAII）。Android 端实现了 Android KeyStore + AES-256-GCM 加密、日志脱敏、禁止备份等良好安全实践。**主要风险集中在第三方库 civetweb（大量遗留安全问题）和 FFmpeg 命令构造上。**
