## 2024-04-06 - Preview Page Destructive Action Protection & Feedback
**Learning:** The SplashPage acts as an interstitial blocker when the backend API (proxied at 8080) is down. This causes test scripts attempting to direct-navigate to `/#/preview` to time out waiting for elements, because they are stuck on the splash screen which displays a connection error. Furthermore, several critical form actions on the preview page (clear DB, enroll) were executing without any visible toast feedback, leaving users guessing the result of their action.
**Action:** Always include a step to click '继续进入' (Continue) on `/#/` to bypass the splash screen in Playwright scripts when testing without a backend. Always wrap destructive actions (like "Clear DB") in a `Popconfirm` and ensure all asynchronous action buttons present success/error feedback via `message` toasts.
