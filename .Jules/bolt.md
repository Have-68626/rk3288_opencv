## 2024-05-24 - Relocated Data Integrity Checks from Hot Loops to Module Boundaries
**Learning:** Checking for edge-case mathematical faults (`std::isnan`) deep inside critical performance loops degrades throughput and disrupts branch prediction. Since index contents and incoming queries are immutable over the duration of a search, the same mathematical safety can be guaranteed by hoisting the checks to the data ingestion boundaries.
**Action:** When optimizing loop invariants in C++, check if boundary validation (e.g., verifying vectors in `reset` or prior to entering loops) can fully replace repetitive inner-loop validation.
